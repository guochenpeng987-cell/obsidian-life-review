# ChatGPT 夜审 / 周审提示词

## 夜审提示词

请连接我的 GitHub 仓库，从根目录 `README.md` 开始，再重点阅读 `_audit/current/` 下的审计文件。

请按以下顺序检查：

1. 目录层级是否清晰，是否有重复层、空层、职责混乱  
2. 文件命名是否统一，是否存在歧义、过长、重复命名  
3. 各领域、项目、知识、人物、系统模块是否失衡  
4. 今天新增或修改的笔记，是否放在了正确位置  
5. 哪些内容更适合移动到别的模块  
6. 哪些笔记过空、过重、重复、可合并  
7. 是否存在孤儿笔记、断链风险、空模板残留  
8. 给出“今晚可做的 3 个最小优化动作”

请按这 4 个栏目输出：

- 结构问题
- 内容问题
- 建议移动 / 合并的文件
- 今晚就能做的 3 个动作

---

## 周审提示词

请基于当前 GitHub 仓库内容，做一次个人运营台周审。

重点检查：

1. 哪些一级模块过重或过轻  
2. 哪些领域长期没有维护  
3. 哪些项目应该完成、暂停或拆分  
4. 哪些知识笔记没有形成主题 / 长青 / 方法升级  
5. 哪些系统规则与实际使用脱节  
6. 给出下周最值得优化的 5 个点，并按优先级排序

请按以下栏目输出：

- 本周结构状态
- 本周内容状态
- 下周最值得优化的 5 个点
- 推荐的 1 个小重构动作

---

## 使用建议

- 每晚优先看 `_audit/current/03_changed_notes.md`
- 每周再结合 `00_tree.md + 01_inventory.csv + 02_core_notes.md` 做全量复盘
- 避免要求一次性重构整个系统，优先做“小步高收益”优化
