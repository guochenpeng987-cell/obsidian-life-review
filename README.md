# 个人运营台审阅仓库

这个仓库不是日常工作仓库，而是 **Obsidian 个人运营台的审阅镜像**。

它的作用是：

- 保留目录层级
- 给 ChatGPT 提供可持续审阅的 Markdown 仓库
- 通过 `_audit/current/` 提供更容易理解的审计视图
- 让日审 / 周审稳定运行

---

## 建议阅读顺序

请优先阅读以下文件：

1. `_audit/current/00_tree.md`
2. `_audit/current/01_inventory.csv`
3. `_audit/current/02_core_notes.md`
4. `_audit/current/03_changed_notes.md`
5. 如有需要，再回到原始笔记查看上下文

---

## 一级目录说明

- `00 收件箱`：临时接住内容，不做长期存放
- `01 日记`：每日记录与周期回顾
- `02 领域`：长期责任区与维持标准
- `03 项目`：阶段性推进事项
- `04 知识`：摘录、主题、长青、方法
- `05 人物`：关系管理
- `06 系统`：模板、导航、规则、说明
- `99 归档`：已结束或低频内容

---

## 审计文件说明

### `_audit/current/00_tree.md`
用于保留完整目录树，方便检查层级是否合理。

### `_audit/current/01_inventory.csv`
用于查看全库台账，包括路径、类型、修改时间、字数估算、标签、链接数量等。

### `_audit/current/02_core_notes.md`
用于查看系统骨架，包括：
- 首页
- 各总览页
- 各领域主页
- 当前规则页
- 核心系统说明

### `_audit/current/03_changed_notes.md`
用于查看最近 24 小时修改过的笔记，便于夜审增量变化。

### `_audit/current/04_link_health.csv`
用于查看结构健康，如：
- 孤儿笔记
- 断链
- 空笔记
- 重复标题

---

## 注意事项

1. 这个仓库是 **只供审阅** 的镜像，不建议在这里直接写作。
2. 如果发现敏感内容不该出现在仓库，请在 `config.json` 的排除项里加入规则。
3. 如果你的 Vault 结构发生大改，请重新运行脚本并提交一次全量更新。
